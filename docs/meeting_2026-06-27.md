## Zielbild der Anwendung

- Einsatz an der Stelle wie auf den Bildern: Kartoffeln laufen über Rollen.
- Use Case ist die manuelle Qualitätskontrolle vor Auslieferung an den Kunden; betrachtet werden Kartoffeln, die schon länger liegen.
- Kamera von oben; jede Kartoffel dreht sich auf der Rolle um die eigene Achse (teils zusätzliche Winkel).
- Erster Schritt: nur Objekterkennung, noch keine Sortierentscheidung.
- **Entscheidung:** False Negative ist schlimmer als False Positive → entsprechend bei der Metrikwahl gewichten.

## Zielklassen und Labels

- Object Detection auf 4 Labels: `bad`, `cut`, `stone`, `potato`.
- Daraus 3 finale Klassen: `good potato`, `bad potato`, `stone`.
- Mapping-Logik:
  - `bad` mit Confidence > 60 % → `bad potato`.
  - `cut` → `bad potato`.
  - `bad` nur, wenn auch `potato` erkannt wurde; keine `bad`-Detection auf Steinen.
  - Rahmendaten (Größe) prüfen: zu kleine Objekte nicht als `bad` werten.

## Reale Verteilung / Annahmen

- Anteil von guten Kartoffeln, Cuts, schlechten Stellen und Steinen wird angenommen (Schätzung statt exakter Erhebung).
- **Annahme:** Weitere Fehlerquellen (z. B. schrumpelige Kartoffeln) werden wegen begrenzter Kamera-Hardware nicht betrachtet.
